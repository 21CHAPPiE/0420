在 **Agno** 里，要让输出结果“规范化”，核心做法不是靠 prompt 硬约束，而是直接用 **Structured Output**：给 Agent 或 Team 指定一个 **Pydantic schema / JSON Schema**，让最终返回严格匹配该结构。Agno 文档明确说，这样可以避免字段缺失、枚举值非法、格式不一致等常见问题；对支持的模型，Agent 默认就会走这种结构化输出能力。([docs.agno.com][1])

## 最直接的实现方式

### 1. 用 Pydantic 定义输出模型

```python
from pydantic import BaseModel, Field
from typing import Literal

class ForecastResult(BaseModel):
    city: str = Field(description="城市名")
    level: Literal["low", "medium", "high"] = Field(description="风险等级")
    score: float = Field(ge=0, le=1, description="0到1之间的风险分数")
    summary: str = Field(description="简要结论")
```

这里的作用有三层：

* **字段固定**：只能返回 `city / level / score / summary`
* **类型固定**：`score` 必须是 `float`
* **取值固定**：`level` 只能是 `low / medium / high`

这类约束正是 Agno 结构化输出的主要用途。([docs.agno.com][1])

### 2. 在 Agent 上挂 `output_schema`

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat

agent = Agent(
    model=OpenAIChat(id="gpt-4.1"),
    instructions=[
        "根据输入内容做风险判断",
        "不要输出额外解释",
    ],
    output_schema=ForecastResult,
)
```

Agno 的 Agent 结构化输出文档说明：给 agent 设置 `output_schema` 后，返回的不是自由文本，而是一个经过校验的结构化对象。([docs.agno.com][2])

### 3. 调用后直接拿结构化结果

```python
result = agent.run("北京未来24小时有强降雨和大风，请评估风险")
print(result.content)
```

你应当把它当成“**已校验的数据对象**”来处理，而不是再去正则解析文本。Agno 文档对这一点的定位很明确：目标就是“不要再解析自由文本，而是直接得到验证后的对象”。([docs.agno.com][2])

## 想要“更稳”，不要只做 schema

在工程里，“规范化输出”通常要做成 **四层约束**：

### 第一层：Schema 约束

这是最基础的一层，用 `Pydantic` 限定字段、类型、枚举、范围。

例如：

```python
class TaskResult(BaseModel):
    task_id: str
    status: Literal["success", "failed", "partial"]
    confidence: float = Field(ge=0, le=1)
    errors: list[str] = []
```

### 第二层：提示词约束

虽然 schema 已经很强，但仍建议在 instructions 里补充业务规则：

```python
instructions = [
    "输出必须符合给定 schema",
    "confidence 必须是 0 到 1 之间的小数",
    "如果信息不足，status 设为 partial，不要编造",
    "errors 只写真正的问题，不要重复 summary",
]
```

这层主要解决“**值语义正确**”，不是只解决“JSON 格式正确”。

### 第三层：业务后校验

Schema 只能校验“结构”，很多“业务规范”还需要自己写：

```python
def validate_business_rules(data: TaskResult):
    if data.status == "success" and data.errors:
        raise ValueError("success 状态下 errors 应为空")
    return data
```

### 第四层：失败重试 / 降级

即使 Agno 结构化输出已经比自由文本稳定得多，线上仍建议准备：

* 一次结构化重试
* 默认值填充
* 降级到人工兜底 / 普通文本告警

## Team 场景怎么做

如果你是多 Agent 协作，Agno 也支持给 **Team** 设置结构化输出。文档说明 Team 的“最终输出”可以被约束成一个 Pydantic 模型，由 team leader 汇总成员结果后生成结构化对象；也就是说，**结构化的是最终结果，不是每个中间 Agent 输出都自动结构化**。([docs.agno.com][3])

示意：

```python
from agno.team import Team

team = Team(
    members=[agent_a, agent_b, agent_c],
    output_schema=ForecastResult,
)
```

这适合：

* 多源信息汇总
* 多步骤分析后只需要一个统一结果
* 给前端 / API 返回固定格式

([docs.agno.com][3])

## 实战里最推荐的规范化模板

你可以直接按这个模式做：

```python
from pydantic import BaseModel, Field
from typing import Literal
from agno.agent import Agent
from agno.models.openai import OpenAIChat

class StandardOutput(BaseModel):
    code: Literal[0, 1] = Field(description="0表示成功，1表示失败")
    message: str = Field(description="结果说明")
    data: dict = Field(description="核心业务数据")
    trace_id: str = Field(description="调用链追踪ID")

agent = Agent(
    model=OpenAIChat(id="gpt-4.1"),
    instructions=[
        "严格按 schema 返回",
        "data 中只放业务字段，不要混入解释文字",
        "失败时 code=1，并在 message 中说明原因",
    ],
    output_schema=StandardOutput,
)
```

这样你在系统里就能统一处理：

* `code`：程序判断成功失败
* `message`：给日志和界面展示
* `data`：给下游系统消费
* `trace_id`：便于排障

## 常见误区

### 1. 只写 prompt，不写 schema

这是最不稳的。
“请输出 JSON” 和 “真正按 schema 强约束”不是一回事。Agno 推荐的就是直接提供 schema 来拿严格结构化结果。([docs.agno.com][1])

### 2. schema 太松

例如把所有字段都写成 `str`，那看起来结构化了，实际上没有约束力。
应尽量使用：

* `Literal`
* `Enum`
* `Field(ge=..., le=...)`
* 嵌套子模型
* 必填 / 可选字段区分

### 3. 把展示文本和业务数据混在一起

例如：

```json
{
  "result": "判断成功，风险较高，建议……"
}
```

这不利于后续程序消费。应拆成：

```json
{
  "level": "high",
  "score": 0.82,
  "summary": "未来24小时有较高风险"
}
```

### 4. 不做业务后校验

schema 只能保证“像样”，不能保证“业务真对”。

## 一句话结论

在 Agno 里保证输出结果规范化，标准做法是：

**`Pydantic/JSON Schema + output_schema + 明确 instructions + 业务后校验 + 失败重试`**

其中最关键的是 **`output_schema`**，因为这一步才真正把“自由文本”变成“可校验、可消费的结构化结果”。Agno 官方文档对 Agent 和 Team 都提供了这套能力。([docs.agno.com][1])
